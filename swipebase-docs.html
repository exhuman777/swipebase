About SwipeBase

SwipeBase is a lightweight, privacy-first tool that lets you review any dataset using a Tinder-like swipe interface. Upload a CSV, JSON, or Markdown file, swipe through items, and export your decisions.

Why SwipeBase?

  • No account required - Start immediately, no sign-up friction
  • 100% client-side - Your data never leaves your browser
  • Works offline - Once loaded, no internet needed
  • Universal - Works with any CSV, JSON, or Markdown data
  • Mobile-friendly - Swipe gestures work on touch devices

How It Works

  1. Upload - Drop your CSV, JSON, or Markdown file (or try a demo)
  2. Swipe - Review items one by one: Yes, No, or Maybe
  3. Export - Download your decisions as CSV, JSON, Excel, or Markdown

File Formats

CSV Format

Standard CSV with headers in the first row. UTF-8 encoding recommended.

name,type,amount,description,url
"AI Grant",grant,"$80K","No-strings credits",https://aigrant.com
"ETHGlobal",hackathon,"$500K","Ethereum hackathon",https://ethglobal.com

JSON Format

Array of objects, or an object with a data array.

[
  {
    "name": "AI Grant",
    "type": "grant",
    "amount": "$80K",
    "description": "No-strings credits",
    "url": "https://aigrant.com"

Technical Details

Architecture

SwipeBase requires no backend or API. It's a single HTML file with embedded CSS and JavaScript that runs entirely in your browser.

No Server Required

All file parsing, data processing, and export generation happens client-side using native browser APIs (FileReader, Blob, URL.createObjectURL).

Technologies Used

  • HTML5 - Semantic markup
  • CSS3 - Custom properties, flexbox, grid, animations
  • Vanilla JavaScript - No frameworks, no dependencies
  • FileReader API - For reading uploaded files
  • Blob API - For generating export files
  • Touch Events - For mobile swipe gestures

Browser Support

  • Chrome 60+ (recommended)
  • Firefox 55+
  • Safari 12+
  • Edge 79+
  • Mobile Safari (iOS 12+)
  • Chrome for Android

Self-Hosting

SwipeBase is a single HTML file. To self-host:

  1. Download swipebase.html
  2. Place it on any web server or open locally
  3. No build step, no configuration required

Customization

Edit the CSS variables at the top of the file to customize colors:

:root {
    --bg-1: #0a0a0a;     /* Main background */
    --text-0: #ffffff;   /* Primary text */
    --yes: #00d26a;      /* Accept color */
    --no: #ff4757;       /* Reject color */
    --maybe: #3b82f6;    /* Maybe color */
}

Frequently Asked Questions

Is my data sent to any server?
No. SwipeBase is 100% client-side. Your files are processed entirely in your browser and never uploaded anywhere. We have no backend, no database, no analytics that track your content.
Does it work offline?
Yes. Once the page is loaded, you can disconnect from the internet and continue using SwipeBase. All functionality works offline.
What's the maximum file size?
There's no hard limit, but performance depends on your device. We recommend keeping files under 10MB / 10,000 rows for smooth operation.
Can I use it on mobile?
Yes. SwipeBase is fully responsive and supports touch gestures. Swipe left for No, right for Yes, just like Tinder.
How do I save my progress?
Currently, decisions are stored in browser memory during your session. Use the Export feature (CSV, JSON, Excel, or Markdown) to save your Yes/Maybe/No lists before closing the tab.
Can I customize the card layout?
SwipeBase auto-detects columns from your file. The first 4 non-title/description fields are shown as rows. For deeper customization, edit the source code (it's open).
What's the difference between Free and Pro?
Free gives you full swipe functionality with CSV/Markdown export. Pro adds sharing, team features, advanced exports (Notion/Sheets), and removes deck limits.
Can I embed SwipeBase on my website?
Yes. It's a single HTML file. You can iframe it or modify the source code for your needs. Pro users get white-label/custom branding options.
Is there an API?
The current version is frontend-only with no API. Pro tier will include API access for programmatic deck creation and result retrieval.
Can multiple people swipe the same deck?
Coming in Pro: Share a deck link and collect swipe decisions from multiple team members, then see aggregated results.
